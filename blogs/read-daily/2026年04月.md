# 20260420 Hermes Agent
+ [https://hermes-agent.nousresearch.com/docs](https://hermes-agent.nousresearch.com/docs)
+ 一个越用越强的自主智能体
+ 关键特性：
    - closed learning loop（闭合学习循环）
        * Agent-curated memory 代理维护的记忆
        * periodic nudges 周期性提醒
        * autonomous skill creation 技能自主创建
        * skill self-improvement during use 使用过程中改进
        * FTS5 跨会话回忆（结合 LLM 摘要）
        * Honcho辩证用户建模
    - 环境在空闲时会进入休眠状态，几乎不消耗任何资源
    - 支持agentskills.io
    - 完全网页控制
+ Honcho
    - 解决的问题：代理会忘记事情、自相矛盾，或者在会话过程中丢失上下文
    - 问题不在于工程技术，而在于你没有从数据中提取出所有潜在信息。你需要进行穷举推理，处理矛盾，追踪长期的模式，并保持数据的一致性

# 20260418 PI
[https://github.com/badlogic/pi-mono](https://github.com/badlogic/pi-mono)

+ 你应该让 **pi** 来适配你的工作流，而不是反过来让自己迁就它
+ **Pi** 自带很强的默认能力，但它会刻意跳过一些功能，比如子代理和计划模式。
+ 会话以树状结构存储，所有分支保存在一个文件中。按消息类型筛选，将条目标记为书签。
+ **AGENTS.md**：项目指令文件，启动时从 `~/.pi/agent/`、父级目录和当前目录加载
+ 当接近上下文限制时，自动摘要较早的消息。通过扩展完全可定制：实现基于话题的压缩、代码感知的摘要，或使用不同的摘要模型。
+ skills：带有指令和工具的功能包，按需加载。渐进式展示，不会撑爆提示词缓存
+ 动态上下文：扩展可以在每轮对话前注入消息
+ Steer 和 follow up 两种模式的中间发送消息
+ 扩展：提供的是原语，而非固定功能。
