# 20260418 PI
[https://github.com/badlogic/pi-mono](https://github.com/badlogic/pi-mono)

+ 你应该让 **pi** 来适配你的工作流，而不是反过来让自己迁就它
+ **Pi** 自带很强的默认能力，但它会刻意跳过一些功能，比如子代理和计划模式。
+ 会话以树状结构存储，所有分支保存在一个文件中。按消息类型筛选，将条目标记为书签。
+ **AGENTS.md**：项目指令文件，启动时从 `~/.pi/agent/`、父级目录和当前目录加载
+ 当接近上下文限制时，自动摘要较早的消息。通过扩展完全可定制：实现基于话题的压缩、代码感知的摘要，或使用不同的摘要模型。
+ skills：带有指令和工具的功能包，按需加载。渐进式展示，不会撑爆提示词缓存
+ 动态上下文：扩展可以在每轮对话前注入消息
+ Steer 和 follow up 两种模式的中间发送消息
+ 扩展：提供的是原语，而非固定功能。
