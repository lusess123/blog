# 20260318 Improving instruction hierarchy in frontier LLMs 上
+ [https://openai.com/index/instruction-hierarchy-challenge/](https://openai.com/index/instruction-hierarchy-challenge/)
+ 大家对 AI 的终极想象，一直都是我们睡觉时，它也能 24 小时不停地独立工作。
+ 从 **1 月**开始，圈子里又出现了一个很火的概念，叫 **rough loop**，本质上就是用非常简单的循环迭代，强行让模型“工作更久”。
+ **OpenClaw** 代表的是一种“始终在线、长期运行、完全自主”的代理，这和我们以前用过的大多数 agent 系统很不一样。
+ 以前通常还是人类在主导，靠人不断下提示，推动下一步；而 **OpenClaw** 是一直在线的，而且是主动行动的。
+ **OpenClaw** 是第一个真正把 **2026 年**最大范式转移打开给大家看的项目，也就是我们正在从“副驾驶式、按任务驱动的系统”转向“长时间运行的完全自主代理”。
+ 怎样设计一个能跨多个 session、多个 agent 协作运转的系统；怎样设计合适的工作流，让每个 session 都能把相关上下文重新取回来；又怎样选对工具，把模型的能力尽可能榨出来。
+ 三个高层结论：
    - 对于长程任务 agent 来说，系统设计里最关键的是创造一个“可读、可理解”的环境，让每个子代理或新 session 都能迅速知道项目当前进展到哪了。
    - 验证极其重要。
    - 我们要更敢于信任模型本身。
+ 所谓“更信任模型”，意思不是一上来就把大量推理逻辑和业务规则封进专用工具里，而是应该先给模型尽可能多的上下文，再配上它原生就理解的通用工具，让它像人一样去探索、判断和行动
+ 第一个失败模式，就是 agent 特别容易“一口气做太多事情”。

# 20260311 Rebuilding Voice Recognition for the AI Era 中 2
+ [https://www.youtube.com/watch?v=Elm2p_TRPwk](https://www.youtube.com/watch?v=Elm2p_TRPwk)
+ 我学到的最大一课，就是作为创始人，你必须相信自己的直觉。因为真正最懂你所处市场、以及公司该找到什么样“公司与市场匹配”的人，往往还是你自己。
+ 现在还处在这样一个阶段：一开始你根本听不出来，但聊到一半有些人会突然意识到“等等，我居然是在跟 AI 讲话”，于是场面就变得有点奇怪。
+ 不只是机器人，消费硬件以后也会越来越多地通过语音来交互。比如楼下的咖啡机，现在还得一层层点触摸屏；但理论上你完全可以直接开口说出想要的咖啡类型。这就像当年触摸屏花了很久才普及到各种消费设备上一样。
+ 当前最热的方向就是实时语音代理、消费硬件，以及医疗和销售里的环境式持续监听能力，这些地方正在冒出大量新应用，仍是一片很大的蓝海
+ 所谓“更聪明”，指的是模型不只是把语音转成文字，而是还能接受指令、可被控制，并在整体上更有“理解力”。现在的语音 AI 虽然已经很好，但本质上还是有点“笨”。
+ 这里的“笨”主要体现在它们还不够会理解上下文。比如有人在嘈杂房间里跟你说话，旁边还有婴儿在哭，他的语气又很紧张；再比如不同语言的说话人混在一起时，模型还不够擅长判断谁是主说话者、谁只是背景声音。
+ 如果你在做机器人，一个经典问题就是：到底该听谁的命令，谁的声音又只是背景闲聊
+ 这个模型定位在多模态 **LM** 和传统语音转文字模型之间。重点不是把它做成一个万能助手，而是做成一个足够可靠、能稳稳待在“转录及其相邻任务”轨道上的模型，同时又保留接受指令的能力
+ 过去你们做的转录模型只负责转录，本身缺少 **LLM** 那种通用智能，所以大家通常是把转录模型和 **LLM** 组合起来用。而这个新模型`Universal 3 Pro`，像是把二者融合到了一起。它仍然是转录模型，但里面已经注入了 **LLM** 的通用理解能力。
+ 一个通用的多模态 **LM**，转录任务可能只占它全部训练数据的 10%，它还学了数学等各种别的能力。但他们的客户需要的是一个稳定理解语音和说话者的语音 AI，所以模型被刻意聚焦在这些任务上，不会误以为自己是个通用助手，而是老老实实待在这个狭窄但高价值的空间里。
