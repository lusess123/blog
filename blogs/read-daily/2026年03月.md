# 20260309 Rebuilding Voice Recognition for the AI Era 中
+ [https://www.youtube.com/watch?v=Elm2p_TRPwk](https://www.youtube.com/watch?v=Elm2p_TRPwk)
+ 如果你只是盯着一个商业机会，对产品或者这个领域本身并不痴迷的话，你会发现自己慢慢就失去兴趣了。但我觉得你应该选——我的经验教训是，我也会跟其他创始人分享：你应该选一个你真正着迷的问题。
+ 我做 **Assembly** 特别开心。我一直在用我们自己的产品，持续地吃自己的狗粮、不断地体验它、给别人做 demo，真的很有趣。
+ **Assembly** 的故事里最疯狂的部分大概是，你在 2017 年就开始做这件事了，那时候几乎没人对 AI 感兴趣。
+ 我一直有一个信念——这有点像自动驾驶汽车：问题不是"人们会不会想要语音 AI 技术"这种产品市场匹配的问题，而是"它什么时候会达到产品市场匹配？什么时候会变得足够好？"就像自动驾驶一样。
+ 像 **BERT** 这样的 **NLP** 模型，让在转录基础上做摘要或情感分析变得更容易了
+ 早期的用例都不是实时的——比如分析过去一周通过这个呼叫中心的所有通话，找出那些 **NPS** 分数差的，之类的。基本上全是基于预录音频的用例。
+ 大概也就是在最近 18 个月内，实时模型才开始跨过一个新的门槛。
+ 突然之间这些技术就会跨过某个门槛——而且那个门槛到底在哪里真的很难确切指出——但突然它们就跨过去了，产品市场匹配就达到了，然后一飞冲天。我们第一次经历这种感觉是在 2021 年左右，而现在我们正在再次经历，而且是实时和非实时两个方向都在爆发。

# 20260308 Rebuilding Voice Recognition for the AI Era 上
+ [https://www.youtube.com/watch?v=Elm2p_TRPwk](https://www.youtube.com/watch?v=Elm2p_TRPwk)
+ **Granola** 或 **Fireflies** 这类笔记工具，它们背后其实都在使用 **Assembly AI** 的平台
+ zoom ,2017 年就切进 **AI**
+ Dylan 说，自己在大学时就创办过一家公司，不过那并不是一家 **AI** 公司。正是在那段经历里，他自学了编程。他还补充说，现在有了 **AI**，也许你都不需要再走这条路了，但在当年，他是真的去买了一堆关于 **PHP**、**Python** 和 **Django** 的编程书来啃。
+ 2015 年，属于神经网络浪潮非常早期的时候，差不多也是 **ImageNet** 被攻克后的那一段时间。在那个时间点他看清了两件事。
    - 第一件就是，他意识到 **AI**、神经网络和深度学习会在接下来的十年里彻底起飞
    - 第二件事，是他在 2015 年买了一台 **Amazon Echo**。
        * 你人在房间另一头也能唤醒它，离麦克风很远它照样工作。
        * 就算电视开着、周围有很多背景噪音，你隔空朝它喊，它仍然能正常工作
        * 最惊人的是，它好用到让 Dylan 开始围绕这个产品形成新的使用习惯，比如设定时器、问天气、放音乐。真正打动他的，不只是“能用”，而是它终于“可靠到可以养成习惯”。
+ 把 **AI** 和深度学习领域正在发生的创新，用来做出更强的 **voice AI** 技术，同时还要让任何开发者都能非常容易地接入和使用它。因为他还清楚记得，自己在 2012 或 2013 年第一次接触 **Twilio** 时，那种“能力突然向个人开发者开放”的震撼感。
+ 很多伟大的公司，一开始并不是冲着“创业公司”去的，而是工程师出于智力兴趣和切身需求，先把自己想要的东西做出来。放到 Dylan 当年创业的时间点，**AI** 还完全不是热门词，甚至你直接说自己做 **AI**，别人还可能觉得像是在讲骗局；那时候更“正确”的说法反而是 **deep learning**。
